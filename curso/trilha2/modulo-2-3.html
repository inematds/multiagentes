MÓDULO 2.3

✓✗ Faça vs Não Faça

As 5 práticas que multiplicam qualidade e os 5 antipatterns que arruinam o time.

6
Tópicos
50
Minutos
Intermediário
Nível
Reference
Tipo
1

📁 FAÇA: 1 dono por arquivo

Cada arquivo crítico tem um único dono. Outros podem ler, mas só o dono escreve. É a regra que mais previne 'trabalho perdido'.

📌 Como definir territórios

Pastas-âncora por papel.

  • •Backend → src/api/
  • •Frontend → src/ui/
  • •QA → tests/
  • •Docs → docs/

⚠️ Sintoma de violação

git diff mostra commits do mesmo arquivo de 2 teammates diferentes. Overwrite é silencioso.

2

🎯 FAÇA: entregáveis observáveis

Cada role entrega artefato verificável: arquivo, endpoint, teste passando — não 'um relatório'. Artefato força o agente a fechar.

✓ Entregáveis observáveis

  • ✓Arquivo no path X
  • ✓Endpoint que responde
  • ✓Teste que passa
  • ✓Doc que renderiza

✗ Entregáveis vagos

  • ✗'Um sumário'
  • ✗'Análise da situação'
  • ✗'Recomendações'
  • ✗'Avaliação do risco'
3

📨 FAÇA: nomeie destinatários

Inclua nome explícito no prompt: 'Quando terminar, mande para Frontend Dev'. Sem destinatário, todos retornam ao lead, criando gargalo.

📌 Padrão de handoff

Quem fala com quem é parte da arquitetura.

  • •Backend → manda contrato para Frontend
  • •Frontend → manda screenshot para QA
  • •QA → manda relatório para Lead
  • •Security → manda findings para Backend

💡 Lead como árbitro, não middleman

Lead intervém em desempates e síntese final. O resto é P2P.

4

🔢 FAÇA: 3-5 integrantes

3-5 é o doce-spot validado pela doc oficial e pela prática. Acima disso, retornos diminuem rápido. Coordenação adiciona overhead exponencial.

📊 Heurística de tamanho

  • 3 teammates: setup mínimo viável
  • 4-5: ponto ótimo para cross-layer
  • 6+: provavelmente split em 2 teams
  • 10+: swarm; quase nunca compensa

💡 Regra de tasks

5-6 tasks por teammate. Se você tem 15 tasks, comece com 3 teammates.

5

📚 FAÇA: contexto completo no spawn

Inclua links, arquivos, severidade, formato do report — direto no prompt do papel. O que não estiver no spawn, não existe para o teammate.

📌 Anatomia de spawn rico

Tudo que ele precisa para não voltar perguntando.

  • •Pasta de trabalho
  • •Arquivos de referência
  • •Severidade desejada
  • •Formato do entregável
  • •Critério de 'pronto'

⚠️ Erro frequente

Achar que CLAUDE.md cobre tudo. Cobre o global; instruções específicas vão no spawn.

6

⚠️ Os 5 antipatterns

5 erros comuns: arquivo compartilhado, deliverable vago, presumir o plano, 10+ integrantes, nenhum contexto. Reconhecer cada um por nome ajuda a corrigir prompts.

✓ Faça

  • ✓Defina donos
  • ✓Defina entregáveis
  • ✓Nomeie destinatários
  • ✓3-5 integrantes
  • ✓Dê contexto completo

✗ Não faça

  • ✗Compartilhar mesmo arquivo
  • ✗Entregáveis vagos
  • ✗Pressupor o plano
  • ✗10+ integrantes
  • ✗Sem contexto prévio
